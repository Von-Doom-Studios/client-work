# Positioning — BRIJ

---

## One-Line Position

**BRIJ is the agent layer for Solana** — infrastructure that makes autonomous software an economic participant.

For developers: **The OpenRouter for Solana.** One integration. Any protocol. Machine-native.

For consumers (via Travel): **Your AI agent books flights. You pay per call.**

---

## The Category

BRIJ is creating a category: **agent infrastructure**.

Not a wallet. Not a payment gateway. Not an AI framework. The layer between what an agent can reason about and what it can actually do with money on Solana.

When you create a category, you own the definition. Everyone who enters after plays on your terms.

---

## Competitive Landscape

### Direct Competition: None

No one has shipped a unified agent-native economic stack on Solana. Individual pieces exist — wallets, payment protocols, aggregators — but no one has unified Read + Write + Pay + Onramp into a single runtime.

### Adjacent Players

| Player | What They Do | Why They're Not BRIJ |
|--------|-------------|---------------------|
| **Phantom / Solflare** | Human-facing Solana wallets | Built for humans. Manual signing. Browser extensions. Not agent-native. |
| **Coinbase MPC Wallets** | Programmatic wallet infrastructure | No x402. No unified read/write layer. No consumer product. |
| **Stripe** | Payment infrastructure | Human-shaped. Merchant accounts. Card networks. Cannot settle agent-to-agent transactions. |
| **LangChain / CrewAI** | AI agent orchestration | Reasoning layer only. No payment, no wallet, no on-chain capability. |
| **Duffel** | Flight booking API | Traditional API with monthly contracts. Not agent-native. Not x402. BRIJ uses Duffel upstream. |
| **Google Flights / Kayak** | Flight search for humans | Browser-based. No API access. No agent integration. Cookie-based price manipulation. |
| **OpenRouter** | LLM routing | Closest analogy to BRIJ's model — but for LLMs, not on-chain services. BRIJ is the OpenRouter for Solana. |

### The Moat

1. **x402 is a protocol, not a feature.** First-mover advantage on a payment protocol is durable.
2. **Unified stack creates lock-in through convenience.** One integration vs. stitching together 6+ tools.
3. **Travel generates revenue and proves the thesis.** Unlike most infra plays, BRIJ has a consumer product from day one.
4. **200+ developers in early access.** Network effect building.

## Positioning by Product

### BRIJ Platform
**For:** Developers building autonomous agents on Solana
**Position:** The OpenRouter for Solana. One integration. Any protocol. Machine-native.
**Against:** Stitching together indexers, SDKs, wallets, and billing systems
**Key stat:** 200+ in early access. 1,000+ developers on waitlist. 8 APIs unified into one runtime.

### BRIJ Travel
**For:** Anyone who books flights (phased: geeks → early adopters → mainstream)
**Position:** The first flight booking service where your AI agent does the work.
**Against:** Traditional booking — multiple tabs, manipulated prices, opaque fees, wasted time
**Key stat:** Live airline inventory. Real bookings. On-chain settlement. Four API calls from search to confirmation.

### x402 Protocol
**For:** API providers who want agent-native monetization
**Position:** HTTP-native payments. Any API becomes pay-per-call. No billing infrastructure needed.
**Against:** Subscription APIs, API key management, usage-based billing platforms
**Key stat:** BRIJ Travel runs entirely on x402 at production scale.

## Revenue Model

BRIJ makes money two ways:
1. **Fee on x402 settlements.** Every time an agent pays for a service through BRIJ, we take a cut.
2. **Fee on on-ramp/off-ramp conversions.** Every fiat-to-stablecoin conversion routes through our aggregator; we take a margin.

Travel adds a third: the spread between wholesale flight cost and what the agent pays.

No subscriptions. No monthly minimums. Revenue scales with usage.

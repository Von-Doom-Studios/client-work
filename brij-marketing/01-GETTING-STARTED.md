## Ground rules (non-negotiable)

1. **Senior professional bar.** Every artifact should be something a founder, agent, or new hire could pick up and execute the same day. No memos. No frameworks. If it's not actionable, it doesn't belong here.
2. **GitHub is the documentation system.** Not Notion. Not Google Docs. Not Discord threads. Anything that needs to persist lives here.
3. **One file per concern.** Strategy in 02. Calendar in 04. Campaigns in 05. Do not mix. Cross-reference instead.
4. **Every file has a header.** Title, summary, owner, related files, last updated date.
5. **Update on change.** When state changes, the file that owns that state gets updated the same day. Stale files are worse than missing ones.
6. **Voice is dry, technical, confident.** Code, numbers, screenshots over adjectives. Builders trust receipts, not takes. No "excited to announce." No rocket emojis. No em dashes.
7. **No token narrative in marketing.**
8. **X-only.** No LinkedIn, no YouTube, no TikTok, no IG, no Farcaster.
9. **External outreach goes through Ops, not Marketing.**
10. **Visuals go through VD-Creative.** Marketing briefs; Creative produces.

## How to operate this folder in 5 steps

1. Open [02-STRATEGY.md](./02-STRATEGY.md). Understand positioning, audience, targets, and what we don't do.
2. Open [04-CALENDAR.md](./04-CALENDAR.md). See what's posting this week and next, by date.
3. Check [05-CAMPAIGNS.md](./05-CAMPAIGNS.md) for any active campaign that overrides calendar baseline.
4. Pull X post copy from [06-CONTENT-LIBRARY.md](./06-CONTENT-LIBRARY.md). Brief visuals to Creative.
5. Pull newsletter copy from [08-NEWSLETTER.md](./08-NEWSLETTER.md) when an issue is due, and confirm every issue ties to a campaign in 05.
